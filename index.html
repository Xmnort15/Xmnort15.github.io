Introduction¶

For this tutorial we will be analyzing LAPD crime date from 2010 to the present(2022) to see whether Covid-19 has any affect to the number of crimes or types of crimes in Los Angeles. Los Angeles is a very popular location for vacations so hopefully this tutorial could shed some light on whether there could be increased dangers with Covid-19 around. Now onto the first step of the tutorial.

Getting the Data¶

Data collection is the first part of the data science pipeline. There are various means of getting data whether it be creating your own dataset, finding a file online, or scraping the data from a website's HTML. In our case, we're lucky and don't need to scrape the data. We'll instead be grabbing a file of crime data from Los Angeles' public data website https://data.lacity.org/.

If you want to know what web scraping is and why it is a painstaking process, here is a good resource: link

For us to view the data we will store it in a dataframe. A dataframe is like a table where each row is an entry in the table and the columns are the attributes/categories of an entry. To store the data in the dataframe we will first download a csv file off the website and use pandas.read_csv() to convert this csv file into a dataframe. In this scenerio the LAPD have 2 different datasets for 2010-2019 and 2020-Present so we'll have to create 2 dataframes.

Los Angeles crime data by the LAPD: 2010-2019 and 2020-Present

Processing the Data¶

The next step of the data science pipeline is processing the data, otherwise known as data wrangling. Most of the time the dataframe we get from reading a file will have issues. So in this step we will be fixing up the dataframe so that it will be easier to understand and easier to analyze. Something important to know is often times this is the step that requires the most time for a data scientist. Don't worry if you feel like you've been at this step for a long time.

More info on data wrangling at: link

We'll start with cleaning our LAPD crime dataframe.¶

One of the first things to do is to see if our dataframe contains any missing or duplicated data in its entries. Of course, if you won't be using an attribute in your dataframe that has missing values then there's not much of a reason. With our crime dataframe, we want to first check if there are duplicates so we need to use something that can be unique to each crime. It just so happens that each crime has a unique 'DR_NO' so we can count how many of these unique 'DR_NO' values are in the dataframe and compare it to the number of entries in the dataframe.

After reading in the csv file, the dataframe we get sets dates as Strings rather than actual dates. In order to analyze and graph by date we'll need to convert these strings into datetime objects. The added parameter errors='coerce' in pd.to_datetime() changes any value that can't be converted to a date variable to be NaT (not a time). We then use the method dropna() based on the 'DATE OCC' attribute to remove any entries that contain these NaT from our dataframe.

In order to be converted to a datetime object we'll have to first convert the time from an integer to a string then we can convert the string into datetime. However, we must first format the string so that it can be accepted by the pd.datetime() function. We do this by making every converted string into the same length by filling them with leading zeroes until they have a length of 4. Then add a semicolon in the middle of the string to separate the numbers that correspond to hours from the minutes.

Looks like there's no missing or duplicate values in our covid-19 dataframe. However, the covid cases data from the New York Times only records the cumulative number of covid cases. So if we want to view the number of cases that occur each day we'll have to subtract the cumulative number of covid cases from the day before with the cumulative cases today.

In [13]:
# renames the columns 'cases' into 'cumulative cases' and 'deaths' into 'cumulative deaths'
covid_df.rename(columns = {'cases': 'cumulative cases', 'deaths': 'cumulative deaths'}, inplace = True)

# subtract the current day's cumulative cases with the one the day before to get the number of cases on this day
for i in range(1, len(covid_df)):
    covid_df.loc[i, 'cases'] = covid_df['cumulative cases'][i] - covid_df['cumulative cases'][i - 1]

# the loop doesn't take into account the very first entry, but we know that the cumulative covid cases should be the same as the
# first recorded day of covid cases
covid_df.loc[0, 'cases'] = covid_df['cumulative cases'][0]

From the looks of the graph, it's hard to tell if covid-19 had any noticeable affect to the number of crimes that occurred in the year. There was about a 30,000 decrease in the number of crimes from 2019 to 2020. This decrease could be explained by there being less people outside due to covid lockdown, but there's no definitive answer. However, one very interesting trend is that during the years 2012 and 2016 there is a very drastic decrease in the number of reported crimes. What we know about those years is that a presidential election was going in the US. However, this doesn't explain why there is a drastic decrease in crime reports during these years. There could be a variety of reasons for this change:

    1) Less people in Los Angeles to report crimes because they were at the elections
    2) Less police officers to respond to crimes reports and record them
    3) There were simply less crimes that occurred

Also notice how despite 2020 also having a presidential election occurring, there isn't a sharp decrease in reported crimes. These questions are what makes data science so interesting as there could be thousands of answers for why the data does what it does, and you will have to discover the most likely answer through analysis. You will likely need to use other datasets to see if there are any correlations you could find between datasets.

In this case, there isn't any attribute in our dataframe that could help with explaining these trends so we will be moving on to analyze another part of our data.

# counts the percentage that the given list of crimes each occur for a time period
def get_perc_crimes(df, crime_lst):
    crime_num = []
    
    for crime in crime_lst:
        crime_num.append((crime, (df[df['Crm Cd Desc'] == crime].index.size) / df.index.size))
    
    return crime_num

Machine Learning¶

For this phase of the data science pipeline we will be trying various algorithms like Linear Regressions or Polynomial Regressions to hopefully get a predictive model of a trend. A good model will allow us to predict values outside of our current data. In this case we will try to model what percentage of stolen vehicle crimes can occur during a year.

We've noticed that stolen vehicle crimes are incredible common in Los Angeles and often those on vacation will likely rent a car. So creating a model to see the percentage of stolen vehicle crimes in the future may help those wanting to vacation to Los Angeles think about whether it's a good idea.

We'll first calculate the percentage of crimes each year that are stolen vehicle crimes and store them in an array.

Our model is the red line and as we can see it doesn't line up with our data at all. A linear regression is only a straight line and since our data isn't linear, the model of course won't be able to match it. We can even measure how well our line fits the data by using the R^2 value. A R^2 value of 1 means that our model fits the data perfectly while a value of 0 means that it doesn't fit the data at all.

What degree you choose depends on how well the model fits the data and whether it's worth using a higher degree. With a higher degree, the computer will require more time and resources to compute the model.

A R^2 of .846 is much better, but this also creates another problem. With a curve like this it seems as though the percentage of stolen vehicle crimes will increase a lot in the coming years which would be unrealistic. This would be a case of overfitting. Overfitting occurs when your model fits the data you have too well that it doesn't predict anything outside of your current data correctly. So despite the fact that our R^2 for a polynomial regression is good, this model likely isn't very helpful in predicting stolen vehicle crimes outside these set of years.

For more info on overfitting and how to prevent it: link

Based off the shape of our data points something like a bell-shaped curve or sigmoid curve would probably fit the data much better. However, the sklearn library doesn't include models of these types which makes it difficult for us to model and predict curves like those. Insights Attained¶

This what most consider the last part of the data science pipeline and where we try to draw insights from our data analyzed.

After creating our graphs and models there doesn't seem to be a correlation between Covid-19 and the number of crimes that occur or times of crimes.

What we can say with some semblance of confidence is that the types of crimes that occur has changed after Covid-19 arrived. Most notably, stolen vehicles crimes have gone up around 3% compared to before and after covid-19. Our models, while flawed, also show an upward trend in percentage of stolen vehicle crimes. While there's no telling why there are more stolen vehicles, we now know that there is an increase. A suggestion would be to station more police officers at very populated LA parking areas to catch thieves in the act. If you ever go to Los Angeles, be careful about where you park and double check that your car in locked.

In general, our analysis doesn't give a good picture of whether Covid-19 affects Los Angeles crimes. But there are plenty of other means of analyzing our data such as using location of crimes or the perpetrator of crimes in conjunction with Covid-19.
